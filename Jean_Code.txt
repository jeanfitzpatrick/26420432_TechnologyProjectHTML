        <div class="w3-container">
          <p>Modeling and Simulations is a computational modelling technique that uses a variety of different applications across many job types. It’s a method that mimics individual agent behaviour and how they interact with the environment to comprehend system behaviour overall. Agents are variables with distinct traits, behaviours and rules for making decisions. They function inside their immediate surroundings i.e their environment. The behaviour of other agents in the environment in which an agent functions can have an impact on the decisions and actions of the agent. Algorithms, rules and mathematical formulas are used to model agent-environment interactions/simulations. Modelling has many pros.</p>

          <p>It allows for discrete models rather than continuous – many actions are “on” or “off” meaning they either happened or did not happen at all. It's ability to model heterogeneous data – a group that is diverse in nature. There is no need for previous understanding of the aggregate of the phenomenon. It can incorporate randomness into the model. And it allows you to see a model develop over time because the data is generated during the simulation. Although there are many pros, there are also some cons. It is not beneficial when dealing with homogeneous data – group with highly similar characteristics. And it is also computationally expensive.</p>

          <h3 id="plan-and-design"><b>PLAN AND DESIGN:</b></h3>
          <h5><span class="w3-opacity">March 2026</span></h5>
        </div>

        <div class="w3-container">
          <p>The initial idea was to create a model for something to do with online computer and/or remote controlled videogames. I am pretty interested in this topic as I grew up with a PlayStation and Xbox. I have always found the foundations and concept behind video games interesting. I enjoy the coding and the fundamental properties of video games also.</p>

          <p>My idea is for a player to track their progress within a warfare type videogame and for the system to predict how well the player would do on their next round of that game. The system will ask the player to input their statistics from the last five rounds of the game and the model will then incorporate the different aspects of the game to figure out the players progress or regress. It will take which team won that round and the players left on each team into account and finish with how many rounds the player has previously won. The model will tell the player if they have progressed and how much by, or if they have regressed and how much by. The system will then tell the player how likely they are to win the next round.</p>

          <p>I researched modelling and simulations pros and cons and decided on my project. I then drew a mind map showcasing all my initial thoughts for the simulation.           <div class="w3-card-4 w3-margin w3-white">
            <div class="w3-container">
              <h6>Testing:</h6>
            </div>
          </div>

          <p>When testing my simulation, I unit tested my project after every change to make sure everything worked properly. I tested while I was completing my code and changed whatever the problem was whether it was a sign error or an integer change. I firstly tested the conditions based on which team won. This worked well and I then went on to test the conditions based on how many players are left on each team. I initially tested while inputting random numbers as that simulated the different results players would get. The original code did not work as it had too high of a base number in general. I changed the number on the code and it worked perfectly. I then tested using actual data. There were three different game round results I tested on. The first set of data showed the players results and their personal statistics, the second set of data showed the same and so did the final set. The model acted correctly to each data set, and it showed the correct statistics for the inputted data. This showed me that the model was working correctly and was ready for public use.</p>

I struggled with the foundations of the Model and Simulation, but eventually understood as I got some help and figured things out on my own as well. The first couple of problems were syntax errors in my code. Some signs needed changing or some numbers needed to change for the model to act accordingly. One problem I faced was the integer numbers. The base number was originally too high for the inputted data and so I changed it accordingly. I did not figure this out until I debugged the code and found the error. I changed the numbers and it worked perfectly. This was a vital area in my project as I needed to use the numbers that is in the code for my simulation to work correctly. I also faced layout problems with my HTML code and used the Inspect Element to figure out the problem. It was an unclosed div tab. I closed the tab and the layout was perfect. Overall, I'm glad I chose a video game concept as it resonated with me and in the end was a very enjoyable project to complete.</p>
